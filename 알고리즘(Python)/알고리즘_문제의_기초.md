# 파이썬 기준

## 1. 문제 조건을 바탕으로 시간 복잡도(Big-O) 계산

- 2000만 번 연산 -> 1초
- 모든 코딩 테스트는 가장 최악의 시간 복잡도(Big-O)가 기준.
- 문제 해결에 사용되는 알고리즘의 시간 복잡도를 미리 계산하기.


## 2. 디버깅(with VScode)

- 각종 변수의 값 등을 확인함으로써 코더의 의도대로 코드가 짜였는지 검토.
- 종단점을 활용하여 부분 체크가 가능(중요)
- 디버깅 작업은 VScode 좌측 바의 디버깅 버튼을 클릭하여 진행(F5)
- +@ de+bug에서 bug는 벌레가 하드웨어에 들어가서 에러를 일으킨 이후로 통용되는 컴퓨터 에러 관련 용어이다. 그러므로 IDE에서 벌레가 그려진 버튼이 보이면 대체로 디버깅과 관련이 있음.
