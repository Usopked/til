# 파이썬 기준

## 배열의 특징

- 특정 값에 바로 접근이 가능(인덱스 번호)
- 값의 삽입, 삭제가 어려움
- 배열을 선언하면 이후 크기를 조정할 수 없음
- 구조가 단순하고 직관적이라 문제풀이에 자주 기용됨


## 리스트의 특징

- 대체로 배열과 정반대 성격
- 특정 값에 바로 접근이 어려움
- 선언할 때 크기 지정이 필요없고, 삽입과 삭제가 자유로움
- 포인터를 저장할 공간이 필요해 배열보다 구조가 복잡함


## 파이썬의 배열과 리스트

- 파이썬에서는 배열과 리스트를 구분짓지 않음(배열+리스트 형태)
- 포인터를 직접 다룰 일이 없어 구현이 쉽고 알고리즘에만 집중하면 됨


## 구간 합

- 합 배열을 사용하여 특정 구간까지의 합을 확인할 수 있음(예: 피보나치 수열)
- 합 배열 공식: S[i] = S[i-1] + A[i]
- 구간합 공식(i~j까지의 인덱스): S[j] - S[i-1]
- **주의** : S[0] == 0으로 둘 것. 인덱스 범위를 고려하지 않으면 i가 1일 경우 난처해짐(A 배열의 첫 번째 수를 포함시켜야 하므로 공식에 예외처리가 필요함.)
